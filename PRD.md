## 1. 文档概述

### 1.1 背景

个人财务管理日益重要，但现有记账应用普遍存在功能臃肿、学习成本高、广告干扰多等问题。用户需要一个简洁、高效、专注日常收支记录的工具。

### 1.2 目标

- 让用户能在 **3 秒内** 完成一笔收支记录
- 提供清晰的 **周度/月度统计报表**，帮助洞察消费趋势
- 保持 **清爽简洁** 的视觉体验，无花哨动画
- Web 端先行，后续可扩展至移动端

### 1.3 目标用户

| 用户类型 | 描述 | 核心诉求 |
|----------|------|----------|
| 上班族 | 有固定收入和定期支出 | 快速记录、月度预算控制 |
| 学生 | 生活费有限 | 消费分类统计、省钱提醒 |
| 自由职业者 | 收入不固定 | 收入来源分类、现金流分析 |

### 1.4 产品定位

一款 **Web 端优先** 的轻量级个人收支记录与统计工具，强调 **简洁、清爽、高效**，为后续移动端拓展奠定基础。

## 2. 功能需求

### 2.1 模块一：用户认证

#### 2.1.1 注册 / 登录

**功能描述：** 用户可以通过手机号 + 验证码方式注册或登录。

**详细要求：**

| 字段 / 功能 | 说明 | 约束 |
|-------------|------|------|
| 手机号输入 | 11 位中国大陆手机号 | 必填，正则校验 |
| 验证码 | 6 位数字验证码 | 必填，60 秒倒计时 |
| 用户协议 | 登录/注册时勾选同意 | 必须勾选才能提交 |
| 游客模式 | 允许不登录直接使用 | 登录后自动迁移本地数据 |

**交互流程：**

1. 用户访问 App，进入登录页
2. 输入手机号 → 点击获取验证码
3. 输入 6 位验证码 → 点击登录 / 注册
4. 新用户自动注册，老用户直接登录
5. 也可选择"暂不登录，先逛逛"进入游客模式

**验收标准：**

- [ ] 手机号格式不正确时即时提示
- [ ] 验证码 60 秒倒计时，期间按钮禁用
- [ ] 未勾选用户协议时无法提交
- [ ] 登录成功后跳转首页

### 2.2 模块二：收支管理

#### 2.2.1 添加交易记录

**功能描述：** 用户可以快速添加一笔收入或支出记录。

**详细要求：**

| 字段 | 说明 | 约束 |
|------|------|------|
| 金额 | 交易金额 | 必填，正数，保留两位小数，最小值 0.01 |
| 类型 | 收入 / 支出 | 必填，单选切换 |
| 分类 | 交易所属分类 | 必填，从预设分类中选择 |
| 日期 | 交易发生日期 | 默认为当天，可手动选择 |
| 备注 | 补充说明 | 可选，最多 200 字符 |
| 支付方式 | 现金 / 微信 / 支付宝 / 银行卡 | 可选，单选 |

**收入分类预设（6 个）：**

| 分类 | 图标 |
|------|------|
| 工资薪金 | 💰 |
| 奖金 / 提成 | 🏆 |
| 投资收益 | 📈 |
| 兼职收入 | 💼 |
| 退款 / 报销 | ↩️ |
| 其他收入 | 💵 |

**支出分类预设（11 个）：**

| 分类 | 图标 |
|------|------|
| 餐饮美食 | 🍜 |
| 交通出行 | 🚗 |
| 购物消费 | 🛒 |
| 住房租金 | 🏠 |
| 水电缴费 | 💡 |
| 娱乐休闲 | 🎮 |
| 医疗健康 | 💊 |
| 教育学习 | 📚 |
| 通讯费用 | 📱 |
| 人情往来 | 🎁 |
| 其他支出 | 📝 |

**交互流程：**

1. 用户在首页点击"+"按钮，进入录入页
2. 输入金额（数字键盘）
3. 选择收入 / 支出类型（顶部 Tab 切换）
4. 选择对应分类（网格展示）
5. 可选：修改日期、填写备注、选择支付方式
6. 点击"保存"完成录入

**验收标准：**

- [ ] 金额输入框支持小数点，最多两位小数
- [ ] 保存时校验金额 > 0，否则提示错误
- [ ] 保存成功后返回列表页
- [ ] 同一笔记录支持编辑和删除（删除需二次确认）

#### 2.2.2 交易列表

**功能描述：** 展示所有交易记录，支持筛选和搜索。

**详细要求：**

| 功能点 | 说明 |
|--------|------|
| 时间分组 | 按日期分组展示（今天、昨天、本周、本月、更早） |
| 列表项 | 显示分类图标、金额（收入绿色、支出红色）、备注、日期 |
| 筛选 | 按类型（全部 / 收入 / 支出）、按分类、按日期范围 |
| 搜索 | 按备注关键词搜索 |
| 排序 | 默认按日期倒序，支持按金额排序 |
| 分页 | 每次加载 20 条，支持下拉刷新和上拉加载更多 |

**验收标准：**

- [ ] 列表首次加载时间 < 500ms
- [ ] 筛选条件切换后列表实时更新
- [ ] 搜索无结果时展示空状态提示

### 2.3 模块三：统计报表

#### 2.3.1 周报表

**功能描述：** 以周为单位展示收支统计和趋势分析。

**详细要求：**

| 图表 / 数据 | 说明 | 展示形式 |
|-------------|------|----------|
| 本周收支汇总 | 总收入、总支出、结余 | 顶部三个数字卡片 |
| 每日支出趋势 | 近 7 天每天的支出金额变化 | 折线图 |
| 支出分类占比 | 各支出分类的金额占比 | 饼图 / 环形图 |
| 收入来源分布 | 各收入分类的金额占比 | 饼图 / 环形图 |
| 日均消费 | 本周平均每日支出 | 数字卡片 |
| 环比变化 | 与上周相比支出增减百分比 | 带箭头的百分比数字 |

**交互要求：**

- 周报表默认展示「本周」（周一至周日），支持切换至上一周 / 下一周
- 点击饼图的某个分类，可下钻查看该分类下的具体交易明细

#### 2.3.2 月报表

**功能描述：** 以月为单位展示更全面的收支统计和趋势分析。

**详细要求：**

| 图表 / 数据 | 说明 | 展示形式 |
|-------------|------|----------|
| 月度收支汇总 | 总收入、总支出、结余、储蓄率 | 顶部四个数字卡片 |
| 月度支出趋势 | 当月每天的支出金额变化 | 柱状图 |
| 分类排行 | 支出金额从高到低的分类排名 | 横向条形图 |
| 收入趋势 | 当月每天的收入金额变化 | 柱状图 |
| 预算执行率 | 各分类预算使用比例 | 进度条列表 |
| 月度对比 | 与上月同指标对比（收入、支出、结余） | 带涨跌标识的对比卡片 |
| 消费习惯分析 | 最大单笔消费、最常消费分类 | 文字总结 + 数据 |

**交互要求：**

- 月报表默认展示「本月」，支持切换至任意月份
- 柱状图支持点击查看某天的详细数据

#### 2.3.3 自定义报表

**功能描述：** 用户可自定义时间范围和维度生成报表。

**详细要求：**

| 功能点 | 说明 |
|--------|------|
| 时间范围 | 支持自定义起止日期（跨度最大不超过 1 年） |
| 维度选择 | 可按分类、按支付方式等维度统计 |
| 报表导出 | 支持将报表导出为图片分享 |

## 3. 非功能需求

### 3.1 视觉设计

| 要求 | 说明 |
|------|------|
| **主题色调** | 清爽浅色系，白色为主背景，蓝色 (#2f6feb) 作为主色，绿色 (#17a34a) 表示收入，红色 (#dc2626) 表示支出 |
| **设计风格** | 极简、扁平化、无阴影过度装饰 |
| **动画效果** | **无动画效果**，所有交互均为即时反馈，无过渡动效 |
| **字体** | 系统默认字体（-apple-system, "PingFang SC", "Microsoft YaHei"） |
| **圆角** | 统一使用 8px 圆角 |

### 3.2 响应式设计

| 断点 | 屏幕宽度 | 适配策略 |
|------|----------|----------|
| 手机端 | < 768px | 单列布局，紧凑间距 |
| 平板端 | 768px - 1024px | 双列布局 |
| 桌面端 | > 1024px | 最大宽度 1200px 居中布局 |

**设计原则：** 所有页面需同时适配手机、平板、桌面三种尺寸，确保在不同设备上均有良好的使用体验。

### 3.3 性能要求

| 指标 | 要求 |
|------|------|
| 首屏加载时间 | < 2 秒 |
| 列表滚动帧率 | ≥ 55fps |
| 交易保存响应 | < 200ms |
| 报表渲染时间 | < 1 秒 |

### 3.4 安全要求

| 要求 | 说明 |
|------|------|
| 数据传输 | 全链路 HTTPS 加密 |
| 密码存储 | 服务端使用 bcrypt 哈希存储 |
| 权限控制 | 用户只能访问自己的数据 |
| 防注入 | 所有输入参数进行 SQL 注入 / XSS 防护 |

## 7. 页面结构

### 7.1 页面清单

|------|------|--------|------|
| 登录 / 注册 | `/login` | P0 | 手机号验证码登录，支持游客模式 |

| 周报表 | `/reports/weekly` | P0 | 折线图 + 饼图 + 汇总数据 |
| 月报表 | `/reports/monthly` | P0 | 柱状图 + 分类排行 + 预算进度 |
| 个人中心 | `/profile` | P1 | 资料、设置、数据管理 |

### 7.2 导航结构

```
App 布局（侧边栏 + 主内容区）
├── 侧边栏导航
│   ├── 🏠 首页          (/)
│   ├── ➕ 记一笔        (/add)
│   ├── 📊 报表          (/reports/weekly)
│   └── 👤 我的          (/profile)
│
└── 主内容区（根据路由动态渲染）
```

**桌面端布局：** 左侧固定宽度侧边栏（200px）+ 右侧自适应主内容区。

**平板端布局：** 左侧收起态侧边栏（仅图标）+ 右侧主内容区。

**手机端布局：** 顶部 Tab 栏替代侧边栏，内容区全宽。
